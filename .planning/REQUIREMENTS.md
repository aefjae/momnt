# Requirements: Mixmomnt

**Defined:** 2026-03-11
**Core Value:** Sound creators can share the moments that matter — raw and specific — without the pressure of showing their face or releasing a finished product.

## v1 Requirements

### Posts

- [ ] **POST-01**: User can create a post with a photo and caption
- [ ] **POST-02**: User can attach an optional audio clip (MP3 or WAV) to a post
- [ ] **POST-03**: User can create an audio-only post (no photo required)
- [ ] **POST-04**: User can tag a post with mood, vibe, or genre tags
- [ ] **POST-05**: User can edit their own post caption and tags
- [ ] **POST-06**: User can delete their own post

### Feed

- [ ] **FEED-01**: User can view a chronological feed of posts from followed creators
- [ ] **FEED-02**: User can play audio clips inline within the feed without leaving the page
- [ ] **FEED-03**: User can view a discovery feed of posts from all creators
- [ ] **FEED-04**: User can filter/browse feed by tag

### Engagement

- [ ] **ENG-01**: User can react to a post (listen acknowledgement / like)
- [ ] **ENG-02**: User can comment on a post
- [ ] **ENG-03**: User can delete their own comment

### Profiles

- [x] **PROF-01**: User can create a profile with display name and bio
- [x] **PROF-02**: Profile photo is optional — platform works without a face/avatar
- [ ] **PROF-03**: User can view another creator's profile and their posts
- [ ] **PROF-04**: User can follow another creator
- [ ] **PROF-05**: User can unfollow a creator

### Auth

- [x] **AUTH-01**: User can sign up with email and password
- [x] **AUTH-02**: User can log in and stay logged in across sessions
- [x] **AUTH-03**: User can log out
- [x] **AUTH-04**: User can reset password via email link

## v2 Requirements

### Discovery

- **DISC-01**: User can search creators and posts by tag or keyword
- **DISC-02**: User receives personalised recommendations based on listening history
- **DISC-03**: Trending moments surface on an explore page

### Notifications

- **NOTF-01**: User receives in-app notification when someone reacts to their post
- **NOTF-02**: User receives notification when someone follows them
- **NOTF-03**: User can configure notification preferences

### Sharing

- **SHAR-01**: User can share a post via public link
- **SHAR-02**: Shared link opens post with embedded audio player

## Out of Scope

| Feature | Reason |
|---------|--------|
| Full track / album uploads | Moments platform, not a distribution service |
| Video posts | Adds complexity; sound + photo is the format |
| Real-time chat / DMs | Feed-first for v1 |
| Monetisation / tipping | Future milestone after validation |
| iOS / Android native app | Web-first; validate before native investment |
| Live streaming | Out of scope for v1 |
