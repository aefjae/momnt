<task type="auto">
  <name>Task 2: Build js/profile-setup.js — save profile and upload avatar</name>
  <files>js/profile-setup.js</files>
  <action>
    Create js/profile-setup.js. This module handles: auth guard, loading existing profile data, live avatar preview (initials + file), avatar upload to Supabase Storage, and profile upsert.

    **Full logic:**

    ```
    1. INIT (DOMContentLoaded):
       a. Call requireAuth() → returns session or redirects. Store userId = session.user.id
       b. Load existing profile: supabase.from('profiles').select('display_name,bio,avatar_url').eq('id', userId).single()
       c. If profile.display_name exists (returning user): pre-populate form fields and show existing avatar via renderAvatar()
       d. If display_name is null (new user): show '?' initials avatar in amber
       e. Remove page-loading class from body

    2. LIVE INITIALS UPDATE:
       displayNameInput.addEventListener('input', () => {
         if (!currentAvatarFile && !profile.avatar_url) {
           // Update initials preview live as user types
           const initial = (displayNameInput.value || '?')[0].toUpperCase()
           avatarPreview.innerHTML = initial  // Update text inside .avi.avi-am circle
         }
       })

    3. FILE PICKER CHANGE:
       avatarInput.addEventListener('change', (e) => {
         const file = e.target.files[0]
         if (!file) return
         currentAvatarFile = file
         // Client-side preview using FileReader — no upload yet
         const reader = new FileReader()
         reader.onload = (ev) => {
           // Replace initials circle with <img> preview
           avatarPreview.innerHTML = ''
           const img = document.createElement('img')
           img.src = ev.target.result
           img.alt = 'Your avatar preview'
           img.className = 'avi avi-lg'
           avatarPreview.replaceWith(img)
           avatarPreview = img  // Update reference
         }
         reader.readAsDataURL(file)
       })

    4. FORM SUBMIT:
       a. Prevent default, disable button ("Saving...")
       b. Validate: displayName.trim() not empty (show .form-error if empty), maxlength respected by HTML
       c. If currentAvatarFile:
          - Determine extension: currentAvatarFile.name.split('.').pop()
          - Upload: supabase.storage.from('avatars').upload(`${userId}/avatar.${ext}`, currentAvatarFile, { upsert: true, cacheControl: '3600' })
          - On upload error: show error, re-enable button, return
          - Get URL: supabase.storage.from('avatars').getPublicUrl(`${userId}/avatar.${ext}`) → avatarUrl
       d. Upsert profile: supabase.from('profiles').upsert({ id: userId, display_name: displayName.trim(), bio: bio.trim() || null, avatar_url: avatarUrl ?? null })
       e. On upsert error: show error message below form, re-enable button
       f. On success: redirect to /feed.html (page may not exist yet in Phase 1 — that is expected, redirect anyway; a placeholder page can be created or a 404 is acceptable until Phase 3)

    5. CHARACTER COUNT (bio):
       bioInput.addEventListener('input', () => {
         charCount.textContent = `${bioInput.value.length}/160`
       })
    ```

    Anti-patterns to avoid (from research):
    - Do NOT call createClient() in this file — import from ./supabase.js
    - Do NOT use getUser() for the auth check — requireAuth() uses getSession() which is correct for client-side
    - Guard against empty display_name before upload: if display name is empty, do not proceed to storage upload
    - Guard initials against empty string: always use `(value || '?')[0]` pattern

    Error display pattern: show errors in a `.form-error` span inside each `.form-field`, plus a general error div at the bottom of the form for API errors that aren't field-specific (e.g., storage 403).
