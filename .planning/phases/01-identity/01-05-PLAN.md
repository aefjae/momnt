    Run each test in order. Use a real email address you can access for AUTH-01 and AUTH-04.

    **AUTH-01 — Signup creates account:**
    1. Open /signup.html
    2. Enter a real email address + a password (8+ chars) + confirm password
    3. Click "Create account"
    4. Expected: either "Check your inbox" message (email confirmation on) OR redirect to /profile-setup.html (confirmation off)
    5. Go to Supabase Dashboard -> Authentication -> Users — confirm user appears
    PASS if: user appears in Supabase Auth users list

    **AUTH-02 — Session persists across refresh:**
    1. Log in at /login.html with the account from AUTH-01
    2. Complete profile setup (enter any display name)
    3. Open a new browser tab, navigate to /profile-setup.html
    4. Expected: page loads and shows your existing profile (not redirected to login)
    5. Refresh the tab — still logged in
    PASS if: session persists across both new tab and page refresh without logging in again

    **AUTH-03 — Logout clears session:**
    1. While logged in, trigger logout (via the nav logout button or by calling it from browser console: `import('/js/auth-guard.js').then(m => m.logout())`)
    2. Expected: redirected to /login.html
    3. Navigate to /profile-setup.html directly
    4. Expected: redirected back to /login.html (requireAuth() fires)
    5. Check localStorage in DevTools — `sb-*-auth-token` key should be gone
    PASS if: accessing protected page after logout redirects to /login.html

    **AUTH-04 — Password reset via email:**
    1. Open /forgot-password.html (you can be logged out)
    2. Enter your real email address, click "Send reset link"
    3. Expected: success message "Check your inbox..."
    4. Open the email, click the reset link
    5. Expected: lands on /update-password.html with the "Set a new password" form visible (not the loading/expired state)
    6. Enter a new password (8+ chars), confirm it, click "Update password"
    7. Expected: redirected to /login.html
    8. Log in with the new password
    PASS if: can log in with the new password after completing the reset flow

    **PROF-01 — Profile saves display name and bio:**
    1. Log in and go to /profile-setup.html
    2. Enter a display name and bio text, do not upload an avatar
    3. Click "Save profile"
    4. Go to Supabase Dashboard -> Table Editor -> profiles
    5. Expected: row exists with your user ID, display_name and bio populated, avatar_url is NULL
    PASS if: profiles row has correct display_name and bio, avatar_url is null

    **PROF-02 — Platform works without avatar (initials shown):**
    1. Complete profile setup without uploading any avatar (same as PROF-01 above)
    2. On the profile-setup page, while typing your display name, observe the avatar circle
    3. Expected: amber circle showing the first letter of your display name updates live as you type
    4. After saving, if there are any pages that render the avatar (e.g., a navbar), confirm initials appear — not a broken image icon or empty circle
    PASS if: amber initials circle appears and shows the correct initial; no broken image states
